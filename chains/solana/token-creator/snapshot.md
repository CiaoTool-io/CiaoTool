## What is a Solana Token Snapshot?

A Solana token snapshot refers to recording and exporting the addresses of token holders and their respective balances at a specific point in time, used to analyze the current token distribution. Through specialized tools, holding data can be quickly filtered and generated, providing a crucial basis for airdrops, incentives, or market value management.

**Common Uses:**

* Airdrop Whitelists: Filtering eligible token-holding addresses.
* User Tiering: Segmenting core and regular users based on the size of their holdings.
* Distribution Analysis: Understanding holding concentration and the supply distribution structure.
* Precision Marketing: Targeting specific users for campaigns or incentive programs.
* Market Value Management: Assisting in the formulation of market management or release strategies.

### Select Token

Fill in the target token address in the input box, or click the drop-down menu to select a token currently held by your wallet.

### Set Filtering Rules

✅ Top Filter (Optional)

Select the top N holders you want to compile from the preset options:

* Top 100 / 500 / 1000 / 2000 / 5000 / 10000

The system will automatically fetch the top-ranking wallet addresses for that token along with their holding amounts.

✅ Custom Minimum Holdings (Optional)

If you wish to filter out "dust addresses" or set a participation threshold—for instance, keeping only wallets with holdings ≥ 1,000 tokens—you can set a minimum token holding threshold here.

<figure><img src="../../../.gitbook/assets/image (777).png" alt=""><figcaption></figcaption></figure>

### Generate and Export Snapshot

Click \[Token Snapshot], and the system will connect to Solana on-chain data to extract the list of eligible wallet addresses. Once completed, you can choose to export the data in the following format:

* CSV File (Used for importing into airdrop tools)

## Common Failure Cases

* Improper selection of snapshot timing, resulting in low data reference value.
* Failure to filter out exchange or contract addresses, affecting the accuracy of the results.
* Address formatting issues or data export anomalies.
* Using outdated data, leading to strategic deviations.
